7-2 算法竞赛（1）

写代码前

  1. 读题要快
  2. 熟练掌握编辑器或IDE
  3. 不要“霸占”计算机
  4. 减少调试
  5. 互相检查
  6. 使用STL

    写代码中

    ACM题目分类–大类

    1. Ad Hoc，杂题
    2. Complete Search (Iterative/Recursive) ，穷举搜索（迭代/ 回溯）
    3. Divide and Conquer，分治法
    4. Greedy (usually the original ones)，贪心法
    5. Dynamic Programming (usually the original ones)，动态规划
    6. Graph，图论 Mathematics，数学
    7. String Processing，字符串处理
    8. Computational Geometry，计算几何
    9. Some Harder/Rare Problems，罕见问题

      ACM的知识宝库–细分

      •https://www.zhihu.com/question/19719698 •http://blog.csdn.net/ulquiorra0cifer/article/details/7767295
      • IOI大纲：https://people.ksp.sk/~misof/ioi-syllabus/ioi-syllabus.pdf
      •http://hzwer.com/1234.html
      代码规范
      Google C++ 规范：https://google.github.io/styleguide/cppguide.html
      Linux C规范：https://www.kernel.org/doc/html/v4.10/process/coding-style.html

（1）使用万能头文件 #include <bits/stdc++.h>， OJ网站一般都支持，只有poj不支持。
（2）输入判断结尾不要用EOF，用 ‘~’，例如： ~scanf(“%d”, &n)
（3）换行。用 K&R 风格，即左大括号不换行，右 大括号单列一行。
（4）变量定义。变量定义在这个变量被变量定义在这个变量被调用的最近的地方， 例如： for (int i = 0; i < 10; i++) //i只在这个循环体内使用 int s = i * i;
（5）最好不要用宏。不管是宏定义，还是宏函数， 都容易出问题。不要用#define定义常量，而是用 const定义常量，例如： const int MAX = 1000005; 把宏函数写成普通函数。

写完代码后

测试

（1）在程序中加入测试代码

（2）在行命令中重定向
不用在程序中加任何代码。
—例如生成的 的可执行程序是abc，在windows或linux的行 命令中，这样输入和输出到文件：
abc <test.in >test.out

对比测试数据

• 复杂的题目，需要测试情况很多。可写两 个程序，进行“对拍” ：
– 提交到OJ的“好”程序；
– 暴力法程序，目的是用它生成测试数据 。
• 测试的时候，比较两个程序的输出是否一 致。
—例如在windows系统下，生成的可执行文件 分别是abc.exe、abc_1.exe，
用文件比较命 令fc比较它们的输出是否一致。

  2. 八皇后
  3. 并查集
  4. 二叉树
  5. Huffman编码
  6. 最长公共子序列区间DP
  7. 树形DP
